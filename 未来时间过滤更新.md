# 未来时间过滤功能更新

## 问题

提取到的时间是**未来时间**（比当前时间晚超过1天），例如：

```
URL: https://analyse.7m.com.cn/5006126/index_gb.shtml
提取时间: 2026-05-14 07:30:00
当前时间: 2026-05-12

→ 这是比赛开赛时间，不是发布时间！
```

## 解决方案

### 1. 未来时间校验
新增 `is_future_time()` 方法，自动过滤未来时间：
- 如果提取到的时间 > 当前时间 + 1天 → 判定为无效
- 允许1天误差（考虑时区差异）
- 跳过该结果，继续尝试其他提取方法

### 2. 增强赛事页面识别
扩展页面特征评分规则：

| 特征 | 分数 | 说明 |
|------|------|------|
| URL含 `/analyse/` `/match/` `/fixture/` `/vs/` | -20 | 赛事相关路径 |
| 标题含 "VS" / "vs" | -15 | 对阵标识 |
| 标题含 "分析" / "预测" / "analysis" | -10 | 赛事分析类页面 |

## 技术细节

### 时间校验逻辑
```python
def is_future_time(self, dt_string):
    """判断是否是未来时间"""
    parsed = datetime.strptime(dt_string, '%Y-%m-%d %H:%M:%S')
    now = datetime.now()
    return parsed > (now + timedelta(days=1))  # 超过1天判定为未来
```

# 剩余问题深度分析与解决方案

## 案例2: 微博详情页

### 问题描述
```
预期: 2026/5/11 7:56:00
当前提取: 2026-05-05 00:00:00 (错误日期)
```

### 根本原因
**Script中有正确的时间，但被页面文本中的错误日期覆盖了**

1. **Script中的正确数据**（未被提取）:
   ```javascript
   "created_at": "Mon May 11 07:56:02 +0800 2026"
   ```

2. **页面文本中的错误日期**（被正则匹配到）:
   ```
   2026年5月5日
   ```

### 为什么会这样？
- 微博是**单页应用(SPA)**，真实数据在JavaScript变量中
- 我们的`extract_from_regex`在整个HTML源码中搜索，优先匹配到了文本中的日期
- 浏览器渲染后，Script执行生成页面，但我们的正则在Script执行前就匹配了文本

### 解决方案

#### 方案1: 增强Script时间提取（推荐）✅
在`extract_from_jsonld`之后，增加专门的Script变量提取方法：

```python
def extract_from_script_vars(self, soup):
    """从JavaScript变量中提取时间"""
    scripts = soup.find_all('script')
    
    patterns = [
        # 微博格式: "created_at": "Mon May 11 07:56:02 +0800 2026"
        r'"created_at"\s*:\s*"([^"]+)"',
        # 其他常见格式
        r'publishTime\s*[:=]\s*["\']([^"\']+)["\']',
        r'createTime\s*[:=]\s*["\']([^"\']+)["\']',
        r'postDate\s*[:=]\s*["\']([^"\']+)["\']',
    ]
    
    for script in scripts:
        if not script.string:
            continue
        for pattern in patterns:
            match = re.search(pattern, script.string)
            if match:
                return match.group(1), 'script-var'
    
    return None, None
```

然后调整提取方法优先级：
```python
methods = [
    self.extract_from_meta,
    self.extract_from_jsonld,
    self.extract_from_script_vars,  # ← 新增，在regex之前
    self.extract_from_time_tag,
    self.extract_from_data_attributes,
    self.extract_from_class_id,
    self.extract_from_regex
]
```

#### 方案2: 正则提取时排除页面文本
当前问题：正则在整个HTML中搜索，包括页面文本
```python
# 当前代码
text = soup.get_text()  # ← 包含所有文本内容

# 优化后
html_source = str(soup)  # 只在HTML源码中搜索
# 不在纯文本中搜索，避免误匹配
```

### 预期效果
- ✅ 微博的`created_at`会被正确提取
- ✅ 其他SPA应用（React/Vue）的时间也能正确提取
- ✅ 不影响传统网页的提取

## 案例4: 球迷屋新闻页

### 问题描述
```
预期: 2026/5/11 1:59:00
当前提取: 2026-05-10 23:30:00 (偏差1.5小时)
之前观察: 页面显示的是当前访问时间
```

### 根本原因
**页面时间是动态生成的，每次访问都不同**

实际测试：
```
第一次访问: 2026-05-10 23:30 (你之前看到的)
第二次访问: 2026-05-12 21:43 (我现在看到的)
```

### 为什么会这样？
1. **页面没有真实的发布时间字段**
2. **JavaScript动态生成"当前时间"**（可能是页面模板问题）
3. **或者是防爬虫机制**（故意显示假时间）

### 解决方案

#### 方案1: 检测动态时间（推荐）✅
如果提取到的时间与当前时间非常接近（<30分钟），判定为无效：

```python
def is_dynamic_time(self, dt_string):
    """判断是否是动态生成的时间"""
    if not dt_string:
        return False
    try:
        from datetime import timedelta
        parsed = datetime.strptime(dt_string, '%Y-%m-%d %H:%M:%S')
        now = datetime.now()
        diff = abs((now - parsed).total_seconds())
        # 相差小于30分钟，可能是动态时间
        return diff < 1800  # 30分钟
    except:
        return False
```

#### 方案2: 多次验证
对可疑的时间，间隔5秒再次提取，如果两次时间不一致，判定为动态时间。

#### 方案3: 页面评分降权
球迷屋这类页面特征：
- URL含 `/news/数字`
- 时间总是当前时间
- 无meta、JSON-LD等结构化数据

可以在页面特征评分中降低分数，判定为"提取失败"而非"成功"。

### 预期效果
- ✅ 动态时间页面被识别为`no_publish_time`
- ✅ 避免误判当前时间为发布时间

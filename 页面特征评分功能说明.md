# 页面特征评分功能说明

## 更新内容

新增 **页面特征评分** 功能，自动区分"技术失败"和"页面本身没有发布时间"。

## 状态分类（新版）

| 状态 | 说明 | 图标 | 含义 |
|------|------|------|------|
| **success** | 成功提取 | ✓ | 找到了发布时间 |
| **no_publish_time** | 页面无时间 | ⚠️ | 页面本身可能没有发布时间（数据页/列表页/静态页） |
| **extraction_failed** | 提取失败 | ✗ | 页面应该有时间，但提取失败（需要优化提取逻辑） |
| **timeout** | 超时 | ⏱️ | 网络超时或页面加载太慢 |
| **browser_error** | 浏览器错误 | ✗ | 浏览器渲染出错 |

## 评分机制

### 评分规则
- **分数 ≥ 30** → 判定为"应该有发布时间"（extraction_failed）
- **分数 < 30** → 判定为"页面本身没时间"（no_publish_time）

### 正面特征（加分项）

| 特征 | 分数 | 说明 |
|------|------|------|
| `<article>` 标签 | +20 | 标准文章结构 |
| 作者信息 | +15 | 有作者/编辑署名 |
| 正文段落 ≥3段 | +15 | 内容页特征 |
| 评论区 | +10 | 互动型内容 |
| 分享按钮 | +10 | 社交分享功能 |
| 标题含日期 | +10 | 标题中有明确日期 |

### 负面特征（减分项）

| 特征 | 分数 | 说明 |
|------|------|------|
| URL含 `/team/` `/data/` `/results/` 等 | -20 | 数据页路径特征 |
| 表格数量 ≥3个 | -15 | 数据密集型页面 |
| 页面元素 <20个 | -10 | 极简静态页 |
| 数字密度 >30% | -10 | 体育比分/股票数据类页面 |

## 使用示例

### 案例1：体育赛事数据页
```
URL: http://sports.163.com/caipiao/team/football/10094
状态: ⚠️ 页面无时间
评分: -20
特征: -URL含/caipiao/; -3个表格; -高数字密度

→ 判定为数据页，无需发布时间，忽略即可
```

### 案例2：新闻文章页
```
URL: https://news.example.com/article/12345
状态: ✗ 提取失败
评分: 45
特征: +article标签; +作者信息; +5段正文; +分享按钮

→ 判定为内容页，应该有时间但没提取到，需要重试或优化
```

### 案例3：登录限制页
```
URL: https://weibo.com/1234567890/xyz
状态: ✗ 浏览器错误
评分: (无法分析)
特征: (无法分析)

→ 技术问题，需要解决登录/反爬虫
```

## 结果筛选建议

### 需要重试的（高优先级）
```
状态 = "extraction_failed" (评分≥30)
状态 = "timeout"
状态 = "browser_error"
```

### 可以忽略的（低优先级）
```
状态 = "no_publish_time" (评分<30)
```

### 成功提取的
```
状态 = "success"
```

## 界面更新

新增 **"页面特征分析"** 列：
- **绿色徽章（评分≥30）**：页面应该有时间
- **红色徽章（评分<30）**：页面可能没时间
- 鼠标悬停可查看完整特征列表

## 技术细节

- 评分逻辑在 `analyze_page_features()` 方法中
- 每次提取失败时自动触发特征分析
- 分析结果存储在结果的 `page_score` 和 `page_features` 字段中
- 下载的CSV包含完整特征数据，可进一步筛选分析
